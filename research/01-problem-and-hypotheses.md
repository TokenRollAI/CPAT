# 研究问题与假设

## 核心研究问题

> **在一个有限的上下文窗口下，让 agent 主动、可逆地管理自己的上下文（CPAT），
> 相比不管理（ReAct）和被动阈值压缩，能否让它在超长程任务上完成得更好？**

注意限定词：**有限窗口** + **超长程**。这两个条件缺一不可——这正是前五组实验缺失的，
也是 CAT 论文成立的前提。

## 可证伪假设

### H-bounded（本轮主假设）
在受限窗口（W ∈ {32K, 200K}）下的长程研究任务中：
- **ReAct** 会因上下文累积到 W 而**提前终止或退化**（无法纳入新信息）。
- **Threshold-Compression**（超 W 才被动 compact）居中：能继续，但压缩时机/选择粗糙。
- **CPAT**（主动可逆治理）维持**最高任务完成度**，且 token 吞吐可控。

**证伪条件**：若三臂在受限窗口下任务完成度无显著差异，或 CPAT ≤ ReAct，则 H-bounded 被证伪。

### H-reversible（CPAT 独有能力）
存在"先放下、后又需要"的回访任务时，CPAT 的 `payload_offload → restore` 可逆链
让它能用**远超窗口**的信息量完成任务，而 ReAct（要么撑爆、要么永久丢弃）做不到。

**证伪条件**：若 CPAT 在回访任务上 restore 命中率低、或不优于阈值压缩，则 H-reversible 被证伪。

### H-degradation（长程退化曲线，对齐论文 Figure 5）
随着交互轮数增加，ReAct 准确率先升后**饱和退化**（论文：60 轮后），
CPAT 持续维持或上升。

**证伪条件**：若 ReAct 不退化、或 CPAT 同样退化，则 H-degradation 被证伪。

## 已被前五组实验确立的事实（不再重测）

- CPAT 机制正确（8 op、链原子性、可逆吞吐、连环多问均按设计工作）。
- 在**非受限窗口**下，CPAT 省 token（35-57%）但不提升任务质量——**这恰恰因为窗口非受限**。
- ReAct 的 append-only context 在纯累积任务上 cache 反而更稳（这点在受限窗口下可能逆转，待测）。

## 关键变量

| 变量 | 取值 | 作用 |
|---|---|---|
| 硬窗口 W | 32K / 200K | 受限程度（自变量） |
| 臂 mode | react / threshold / cpat | 治理策略（自变量） |
| 任务轮数 | 长程（数十轮） | 长程程度 |
| 信息总量 | > W | 强制治理 |
| 准确率 | F1 / 完成度 | 因变量（质量） |
| 是否提前终止 | bool | 因变量（ReAct 死穴） |
| token 吞吐 | 累计 prompt tokens | 因变量（成本） |
